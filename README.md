# Function Name Prediction from Natural Language

Predict camelCase function names from natural-language descriptions using lightweight ML models designed for embedded deployment.

Given a command like `"start my yoga workout"`, the system returns `startYoga` — mapping human intent to the correct SDK function call.

---

## Problem

When a user issues a voice or text command to an IoT device, the system must map it to the correct execution function. The model must be:

- **Tiny** — under 5 MB to fit on an embedded chip
- **Fast** — sub-millisecond inference on low-power CPUs
- **Offline** — no cloud APIs, no internet
- **Robust** — handle diverse phrasings for the same command

---

## Architecture

```
User Command
     │
     ▼
┌─────────────────────┐
│   OOD Reject Gate   │──── Unknown input? → "Command not recognized"
│  (entropy + gap)    │
└────────┬────────────┘
         │ Known input
         ▼
┌─────────────────────┐
│  Stage 1: Intent    │     6 classes: start_workout, stop_workout,
│  Classifier (LR)    │     live_monitor, get_data, enable/disable_control
└────────┬────────────┘
         │ routes to
         ▼
┌─────────────────────┐
│  Stage 2: Target    │     Per-group sub-classifier with
│  Classifier (LR)    │     group-local TF-IDF vectorizer
└────────┬────────────┘
         │
         ▼
    "startYoga"
```

**Feature Extraction:** Word TF-IDF (bigrams) + Character N-gram TF-IDF (3-6 grams) fused via sparse matrix concatenation.

## Results

| Model | Test Accuracy | Top-3 Accuracy | Latency | Size |
|---|---|---|---|---|
| Flat LR (word only) | 68.57% | 90.00% | 0.08 ms | — |
| Flat LR (word + char) | 98.57% | 100% | 0.10 ms | — |
| **Hierarchical Router** | **100%** | **100%** | ~1 ms | **860 KB** |

- Character n-gram fusion: **+30% accuracy** over word-only baseline
- OOD Reject Gate: **0% false rejection**, **100% OOD detection**
- Dataset: 70 IoT/fitness functions across 6 intent groups, augmented to 210 samples

## Key Design Decisions

| Decision | Why |
|---|---|
| TF-IDF over transformers | BERT = 440 MB. Our model = 860 KB. 100% accuracy either way. |
| Keyword 3x repetition | Keywords contain the discriminative token (sport name). Boosting them amplifies signal above template noise. |
| Hierarchical over flat | 24 start_* descriptions are nearly identical. Flat classifier confuses them. Two-stage routing solves this. |
| Group-local TF-IDF in Stage 2 | Global IDF dilutes sport names. Local IDF correctly identifies them as the most important feature. |
| Char n-grams (3-6) | Handles typos, compound words, and voice transcription errors. "swiming" still matches "swimming". |
| OOD reject gate | On hardware, wrong function execution = safety failure. Better to reject than misfire. |
| Synthetic augmentation | Only 1 sample per function in raw data. Template-based paraphrasing creates 3 variants per class. |
